# Graphs

## Key Concepts

- Represent as adjacency list (dict of lists) or matrix
- BFS for shortest path (unweighted), DFS for exhaustive exploration
- Track visited nodes to avoid infinite loops
- Union-Find for connected components

## Common Patterns

1. **DFS/BFS on grid**: Treat 2D grid as implicit graph, explore neighbors
2. **Topological sort**: Order tasks with dependencies (DAG)
3. **Union-Find**: Group connected components efficiently
4. **Shortest path**: BFS (unweighted), Dijkstra (weighted)
