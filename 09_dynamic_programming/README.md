# Dynamic Programming

## Key Concepts

- Break problem into overlapping subproblems with optimal substructure
- Two approaches: top-down (memoization) or bottom-up (tabulation)
- Start by defining the state and recurrence relation
- Draw out the decision tree, notice repeated work → DP

## Common Patterns

1. **1D DP**: State depends on previous 1-2 elements (climbing stairs, house robber)
2. **2D DP**: Two dimensions of state (strings, grids, knapsack)
3. **Decision making**: At each step choose to include/exclude (knapsack-style)
4. **Interval DP**: Solve for all subintervals
